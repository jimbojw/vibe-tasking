# Vibe Tasking - Documentation Hub

This `docs/` directory serves as the central hub for all documentation related to the Vibe Tasking project. It is organized into subdirectories based on the type and purpose of the documentation.

For definitions of the documentation types (Article, AI-Guide, etc.), please refer to the [Glossary](reference/glossary.md).

## Directory Structure

Below is an overview of the main subdirectories within `docs/` and their purposes:

- **[`articles/`](articles/)**:

  - **Purpose:** Contains user-focused documentation intended for readers. Articles explain concepts, provide guidance on how to use various aspects of the Vibe Tasking framework, or discuss best practices and related methodologies.
  - **Key File:** `articles/README.md` (Introduces the articles section).

- **[`reference/`](reference/)**:

  - **Purpose:** Contains reference materials, such as the [Glossary](reference/glossary.md), technical specifications, or documentation on specific tools and capabilities relevant to the project.
  - **Key File:** `reference/README.md` (Provides an overview of the reference documents available).

- **[`adrs/`](adrs/)**:
  - **Purpose:** Contains Architecture Decision Records (ADRs) that document significant architectural choices made for the Vibe Tasking project.
  - **Key File:** `adrs/README.md` (Explains what ADRs are and how they are used in this project).

Please refer to the `README.md` file within each subdirectory for more detailed information about its contents and conventions.
